## Part A — Install Script

### Your job: think from first principles about what installation means for Claude Code

Before writing code, think through:

1. **What does a user have before running the installer?**
   - The Myna repo cloned somewhere
   - An Obsidian vault somewhere else
   - Claude Code CLI installed (or not — should the script check?)
   - Node.js installed (needed for MCP server)

2. **What does the user need after running the installer?**
   - A CLAUDE.md in their vault (or wherever they run Claude Code) that makes Myna work
   - The obsidian-cli MCP server registered with Claude Code
   - Vault folder structure created
   - Config example files to copy and customize
   - Clear instructions on what to do next

3. **What could go wrong?**
   - Wrong paths, missing dependencies, permission issues, MCP server build failures
   - Running the script twice (idempotent?)
   - User has an existing CLAUDE.md in their vault
   - The MCP server isn't built yet (`dist/` doesn't exist)

4. **What's the user experience?**
   - What do they type to start?
   - What questions does the script ask?
   - What do they see during installation?
   - What do they see at the end?
   - What's the first thing they do after install?

### Starting points for the install flow:

**Step 1: Gather info** — vault path, subfolder name (default: myna), repo path

**Step 2: Create vault folders** — all folders per foundations.md vault structure

**Step 3: Copy config examples** — 6 .example files, don't overwrite existing configs

**Step 4: Generate CLAUDE.md** — from a template with path substitution

**Step 5: Configure MCP** — build server if needed, `claude mcp add`

**Step 6: Verify** — check everything, print summary and next steps

### Your job beyond these starting points:

Think about what ELSE the install script needs:

- Should it check for prerequisites (node, npm, claude CLI)?
- Should it handle the case where the user wants to update an existing install?
- Should it back up an existing CLAUDE.md before overwriting?
- What about the `.example` files — should they go in the repo (versioned) or be generated by the script?
- How should error messages help the user fix problems?
- What's the `--help` output? `--dry-run`? Any other useful flags?
- Should it offer to do the initial `claude mcp add` or just print the command?

### A2. Write the install script

Write `install.sh` in the repo root. Bash. Clear, readable, well-commented. Colored output. Idempotent. Fails loudly.

### A3. Generate config .example files

Create the 6 `.example` config files in `agents/config-examples/`. Generate from schemas in foundations.md. Each file should be:
- Fully commented with explanations
- Have realistic example values (not `<placeholder>`)
- Clearly mark required vs optional fields
- Be valid YAML if copied and edited
- Actually match the schemas described in foundations.md — field for field

### A4. Write CLAUDE.md template

Create `agents/claude-md-template.md`. This is the MOST CRITICAL file in the entire install — it's what Claude Code reads when starting a Myna session.

Think deeply about this template:

- **What must be in CLAUDE.md directly** (always in context, never lost): core identity, critical safety rules, session start instructions, skill activation mechanism
- **What should be read at session start** (loaded into context once): steering files, config files
- **What's read on demand** (loaded when needed): individual skill files
- **Path strategy**: how do you reference agent files from a CLAUDE.md that lives in the vault? Absolute paths? Relative? What if the user moves the repo?
- **Size budget**: CLAUDE.md shouldn't be enormous. What's the minimum needed to make Myna work? What can be deferred to referenced files?
- **The "cold start" test**: a user installs Myna, opens Claude Code in their vault, and says "sync". Will it work? Walk through every step from Claude reading CLAUDE.md to the sync skill executing.

Use `{{VAULT_PATH}}`, `{{SUBFOLDER}}`, `{{REPO_PATH}}` as placeholders the install script substitutes.

> (3) Read the CLAUDE.md template — simulate being Claude Code reading this file. Would you know how to start a Myna session? Is the skill list complete? Are safety rules present? Would 'sync' work?
