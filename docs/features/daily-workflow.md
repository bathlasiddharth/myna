# Daily Workflow — Features

**Scope:** Daily notes, morning sync, quick capture, planning, priority coaching, weekly/monthly summaries, dashboards, review queue, park/resume.

---

## Features

> **Moved to cross-domain.md:** F33 (Park & Resume) — context parking applies across all domains, not just daily workflow.

### Review Queue

One-line summary: Last resort for items the agent can't confidently write — most items bypass the queue via provenance markers.

- **Most items skip the review queue.** The provenance marker system (see non-functional) handles the majority of writes:
  - User-typed → direct write with `[User]` tag
  - Agent, high confidence → direct write with `[Auto]` tag
  - Agent, some inference → direct write with `[Inferred]` tag
  - Agent, genuinely ambiguous (can't determine project, unclear owner, conflicting signals) → review queue
- Four queue files for items that DO need review:
  - `ReviewQueue/review-work.md` — ambiguous action items, delegations, decisions, blockers
  - `ReviewQueue/review-people.md` — ambiguous recognition, observations
  - `ReviewQueue/review-self.md` — contribution candidates from wrap-up that the agent isn't confident about
  - `ReviewQueue/review-triage.md` — items from email triage awaiting user confirmation
- Each entry includes: source reference, agent's interpretation, what's ambiguous and why, proposed action, destination
- Processing flow: approve (write to destination) / edit (modify, then approve) / skip (leave for later) / discard
- Review queue count surfaced in daily note — but the count should be low. If it's consistently high, the agent's confidence thresholds need tuning.

### Morning Sync

One-line summary: "Sync" sets up or refreshes your day by writing a new snapshot to the top of your daily note — rerunnable anytime.

- Creates daily note from template if one doesn't exist for today
- Reads calendar for today's meetings → populates Meetings section with linked prep files
- Creates per-meeting prep files under `Meetings/` for each meeting (with meeting brief content)
- Reads open and overdue tasks → surfaces in daily note
- Reads review queue → shows count
- **Rerunnable throughout the day:** each sync reads any existing snapshots as context (to highlight what changed — new meetings added, tasks completed, queue items resolved since last sync), then prepends a fresh snapshot at the TOP of the daily note with a timestamp header. Previous snapshots stay in place as-is — the agent never moves, collapses, restructures, or deletes them. The user sees the latest state first and manages cleanup of old snapshots.
- Morning Focus section in daily note is user-editable — sync never overwrites it. It lives outside the snapshot sections.
- "Plan tomorrow" creates next day's note the evening before; sync adds overnight updates without overwriting user edits
- Logged with timestamp: "Sync complete (2:30 PM). 4 meetings remaining, 2 overdue tasks, 12 items in review queue."

### Daily Note

One-line summary: Per-day note that serves as your live priority list with Dataview queries and agent-generated sections.

- Sections:
  - **Capacity Check** — available focus time vs. task effort due today, flags over-capacity
  - **Immediate Attention** — auto-generated by morning sync, manually editable throughout the day. Stands out as your live priority list.
  - **Open Tasks** — overdue and due today via Dataview queries
  - **Delegations** — overdue delegations (red flag) and approaching-deadline delegations (next 2 days). This is what prevents things from falling through when you delegate 10+ tasks a week. Queryable via Dataview.
  - **Review Queue** — count of pending items with link to queue files
  - **Milestones** — upcoming birthdays, anniversaries from people files (if enabled)
  - **Today's Meetings** — linked checkboxes to per-meeting prep files
  - **End of Day** — added by wrap-up feature
- Created by morning sync or "plan tomorrow" the evening before
- Live Dataview queries stay current throughout the day without manual refresh


### Weekly Note

One-line summary: Weekly overview showing tasks, deadlines, carry-forwards, goals, and capacity across the week.

- Week Capacity: total meeting hours vs. available focus time vs. task effort estimates
- Flags packed days, suggests rebalancing
- Carry-forwards from previous week highlighted
- Weekly goals section (user-editable)
- Live Dataview queries updated automatically

### Planning

One-line summary: On-demand planning assistance with three modes for different planning needs.

- Three modes:
  - **Plan Day:** looks at calendar, open tasks, priorities → suggests schedule, flags over-commitment, recommends what to tackle first
  - **Priority Coaching:** reviews tasks, meetings, goals → recommends top 3 priorities with reasoning, flags blockers and tasks deferred too many times
  - **Week Optimization:** suggests meetings to skip or delegate, optimal times for focus blocks, what to safely defer
- All output shown inline
- Complements Time Block Planning (meetings-and-calendar domain): Planning decides *what* to work on, Time Block Planning creates the calendar events

### End of Day Wrap-Up

One-line summary: "Wrap up" closes out the day — logs what you did, moves unfinished items, captures final thoughts.

- Writes an "End of Day" section at the bottom of today's daily note containing:
  - **Planned vs actual:** compares the Immediate Attention items from the first morning sync snapshot against what was completed (tasks marked done, meetings checked off, delegations resolved). Shows: completed, not started, partially done.
  - **Unfinished items:** listed explicitly, then moved to tomorrow's daily note (creates it if needed) so they surface in the next sync
  - **Quick notes:** any last thoughts the user wants to capture before closing out
- **Contribution detection:** scans the day's completed work (tasks done, decisions logged in timelines, meetings debriefed, blockers resolved) for items that look like contributions worth tracking. High-confidence contributions (explicitly completed tasks, decisions you clearly drove) written directly with `[Auto]` tag. Less certain ones (agent thinks you might have influenced an outcome) written with `[Inferred]` tag. Genuinely uncertain ones go to `ReviewQueue/review-self.md`. The passive self-tracking loop: you do work all day, wrap-up catches what might be worth remembering.
- The daily note becomes the complete record of the day: sync snapshots at top, user edits in the middle, wrap-up at the bottom
- Feeds data into weekly summary (completed vs carried forward counts)

### Quick Capture

One-line summary: "capture: [anything]" — the agent figures out where it goes, routes it to the right places, and creates a review queue entry for each destination.

- "capture: Sarah did a great job handling the auth incident, and the auth migration is now unblocked" → the agent decomposes this into multiple entries:
  - Recognition for Sarah → Sarah's person file `[Auto]` (user explicitly stated praise)
  - Project update (auth migration unblocked) → auth migration timeline `[Auto]`
  - Your contribution (handled the incident) → contributions log `[Inferred]` (agent interpreted you were involved)
- The agent determines destinations from context — it can be people files, project timelines, task lists, contributions log, personal notes, or any combination
- One capture can produce multiple entries — one per destination. Nothing is silently dropped. Every relevant destination gets its own entry with the appropriate provenance marker.
- Provenance markers determine write path: `[Auto]` for clear items, `[Inferred]` for interpreted ones, review queue only if genuinely ambiguous (can't determine destination)
- If the agent can't determine where something goes, it asks
- Multi-destination routing applies here — see non-functional (Multi-Destination Routing) and cross-domain interaction map for the system-wide principle

### Weekly Summary

One-line summary: On-demand cross-project summary of the week with self-reflection prompts.

- "Weekly summary" → reads any existing summary content in the weekly note as context, then appends a new summary section with timestamp. If run again later in the week, only adds what's new since the last summary — no duplication.
- Covers: key accomplishments, decisions made, blockers hit, tasks completed vs. carried forward
- Self-reflection section: agent-generated prompts based on the week's patterns — time allocation balance, feedback gaps (from people-management), delegation health, recurring carry-overs
- Feeds into monthly updates

### Monthly Update Generation

One-line summary: Generate monthly project updates and business review drafts from accumulated timeline data.

- "Generate monthly update" / "draft my MBR" → compiles from project timelines over the past month
- Output saved to `Drafts/` folder
- Includes trends: month-over-month comparisons, persistent unresolved issues, contribution patterns
- Quarterly reflection available on-demand for deeper 3-month analysis
- Narrative types: MBR (Monthly Business Review), MTR (Monthly Tech Review), QBR (Quarterly)

### Unified Dashboard

One-line summary: Single aggregated view of everything across the vault, always live via Dataview.

- Obsidian-native file at `_system/dashboards/dashboard.md` with live Dataview queries
- Always up-to-date — no refresh needed
- Sections: Immediate Attention, Today's Meetings, Review Queue count, Active Projects with status, Delegation Tracker (overdue/approaching), People Overview (upcoming 1:1s, feedback gaps), Team Health (for managers — see people-management Team Health Overview), Current Drafts with lifecycle state, Recent Activity
